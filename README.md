# rocom-data

从洛克王国世界 BWIKI 爬取完整精灵图鉴数据，包含种族值、技能、特性、属性克制关系，
附带终端图鉴查看器和战斗模拟器（支持自由组队、队伍持久化、图片识别导入）。

> ⚠️ **战斗模拟器仅供参考，结果不代表真实游戏表现。**
> 详见「[模拟器精度说明](#模拟器精度说明)」章节。

## 功能一览

| 功能 | 说明 |
|------|------|
| 🕷️ 全量爬取 | 抓取 BWIKI 上全部精灵数据，保存为 JSON 和 CSV |
| 🔄 检查更新 | 对比本地与 wiki 的差异，仅增量爬取新增精灵 |
| 📖 图鉴查看器 | 在终端按页浏览每只精灵的完整信息，支持搜索 |
| ⚔️ 战斗模拟器 | 自由组队 + 队伍名册 + AI 随机对战，支持双属性和性格配置 |
| 📋 队伍名册 | 队伍持久化存储，内置预设 + 用户自定义，随时调取并编辑 |
| 🖼️ 图片导入 | 识别游戏内队伍分享截图，自动解析精灵和技能并存入名册 |

### 方式一：双击 run.bat（Windows 推荐）

```
================================
  Rocom Sprite Data Tool
================================
  1. Full scrape (all sprites)   ← 全量爬取
  2. Check for updates           ← 检查更新
  3. Browse sprites (viewer)     ← 图鉴查看器
  4. Battle simulator            ← 战斗模拟器
  5. Exit
================================
```

### 方式二：命令行

```bash
python rocom_scraper.py                 # 全量爬取
python rocom_scraper.py --check-update  # 检查更新
python viewer.py                        # 图鉴查看器
python battle.py                        # 战斗模拟器
```

## 爬虫

### 可选参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| `--limit N` | 只爬前 N 只精灵（调试用） | 0（全部） |
| `--delay N` | 每次请求间隔下限（秒），实际为 `delay ~ delay+1.5` 随机值 | 1.5 |
| `--output path` | 输出 JSON 的路径 | `data/sprites.json` |
| `--check-update` | 增量更新模式 | — |

### 输出文件

| 文件 | 说明 |
|------|------|
| `data/sprites.json` | 完整精灵数据，嵌套 JSON |
| `data/sprites.csv` | 扁平化 CSV，可用 Excel / pandas 打开 |
| `data/sprites.backup.json` | 每次爬取前自动备份的上一版 |

### JSON 字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `no` | int | 精灵编号 |
| `name` | str | 精灵名称 |
| `form` | str / null | 形态名，无则为 null |
| `url` | str | BWIKI 原始页面链接 |
| `has_shiny` | bool | 是否有异色图 |
| `attributes` | list[str] | 属性，最多两个，如 `["火", "水"]` |
| `stats` | dict | 种族值：`hp / atk / sp_atk / def / sp_def / spd / total` |
| `ability` | dict | 特性：`name / description` |
| `type_matchup` | dict | 克制关系：`strong_against / weak_to / resists / resisted_by` |
| `skills` | list[dict] | 可学技能列表，见下表 |

**skills 子字段：**

| 字段 | 说明 |
|------|------|
| `name` | 技能名 |
| `attribute` | 技能属性（火/水/草…） |
| `category` | 类别（物攻 / 魔攻 / 状态 / 防御） |
| `power` | 威力数值 |
| `cost` | 能量消耗 |
| `description` | 技能效果描述 |

## 战斗模拟器

```bash
python battle.py
# 或双击 run.bat → 4. Battle simulator
```

### ⚠ 模拟器精度说明

**本模拟器仅供配队思路参考，结果不代表真实游戏中的实际表现。**

| 项目 | 实现情况 | 说明 |
|------|----------|------|
| 六维属性 | ✅ 已实现 | 由种族值 + 个体值规则 + 性格修正公式计算，与游戏公式一致 |
| 双属性精灵 | ✅ 已实现 | 受击克制倍率 = 主属性 × 副属性，221 只双属精灵全覆盖 |
| 性格系统 | ✅ 已实现 | 默认自动分配（种族值最高项+20%，最低项-10%），支持手动覆盖 |
| 技能效果 | ✅ 部分实现 | 通过正则解析技能描述文本，常见效果（buff/debuff/异常/吸血等）已覆盖，复杂描述可能漏解析 |
| 异常状态 | ✅ 已实现 | 烧伤 / 中毒 / 冻结 / 寄生，规则见下方机制表 |
| 应对机制 | ✅ 已实现 | 攻击/防御/状态三类应对的减伤、反弹、威力倍率 |
| 天气系统 | ✅ 已实现 | 雪天 / 沙暴 / 雨天，持续 8 回合 |
| **特性（被动）** | ❌ **未实现** | 特性字段仅作显示，不参与战斗计算 |
| 对战策略 | ❌ 随机 AI | 双方均随机选择可用行动，不代表玩家的实际操作水平 |
| 持久化效果 | ⚠️ 部分实现 | 技能描述中"永久"类效果的触发依赖描述文本解析，准确性因技能而异 |

**特别注意：** 特性是洛克王国的核心玩法之一，每只精灵的特性在游戏内会带来显著的战斗加成（如"溶解扩散"在配毒队时极为关键）。由于特性效果形式多样、难以自动化解析，当前版本尚未实现，因此模拟结果与真实对局存在较大偏差，请以游戏内实际表现为准。

### 编辑精灵配置（管理菜单 → e<序号>）

进入编辑界面后，可同时修改任意精灵的技能和性格，每只精灵旁边显示当前性格：

```
────────────────────────────────────────────────
  编辑精灵配置  ·  队伍「我的速攻队」
  <序号>=编辑技能和性格  n<序号>=仅改性格  0=保存退出
────────────────────────────────────────────────
  1. 影狸      [速度↑魔防↓]   1:嘲弄  |  2:感染病  |  3:毒液渗透  |  4:恶意逃离
  2. 裘卡      [速度↑物防↓]   1:阻断  |  2:崩拳    |  3:毒囊      |  4:防御
  ...
────────────────────────────────────────────────
```

**修改技能**（选序号后进入子编辑）：

```
  当前技能：  1:嘲弄  2:感染病  3:毒液渗透  4:恶意逃离
  输入替换（支持多个，如 1:喷火, 2:水刃，3:冰晶）回车=完成：
  1：毒液渗透，2：感染病，3：恶意逃离，4：嘲弄
```

支持中英文冒号（`:`  `：`）和逗号（`,`  `，`）混用，一行可替换 1~4 个技能槽。

**修改性格**（`n<序号>`）：

```
  影狸 当前性格：速度↑ 魔防↓
  可选属性：生命  物攻  魔攻  物防  魔防  速度
  输入「提升:降低」（如 速度:魔攻），回车=恢复自动[速度↑魔防↓]：速度:物防
  影狸 性格已设为 速度↑ 物防↓  → HP=343 物攻=... 速度=240
```

自定义性格显示带 `*` 标记（如 `[速度↑物防↓*]`）。

### 从图片导入队伍（菜单 5）

识别游戏内标准组队分享截图，自动提取精灵名和技能，存入名册。

**第一步：** 将截图放入 `import_images/` 文件夹。支持 `JPG / PNG / WEBP / GIF`。

**第二步：** 选择识别方式：

| 方式 | 精度 | 费用 | 安装 |
|------|------|------|------|
| Claude Vision API | ★★★★★ | 按量计费 | `pip install anthropic` + 配置 `ANTHROPIC_API_KEY` |
| rapidocr（本地） | ★★★☆☆ | 免费 | `pip install rapidocr-onnxruntime` |
| easyocr（本地） | ★★★★☆ | 免费 | `pip install easyocr`（需 PyTorch） |

**API Key 配置：**

```bash
# Windows CMD
set ANTHROPIC_API_KEY=sk-ant-...
# PowerShell
$env:ANTHROPIC_API_KEY="sk-ant-..."
# 永久：系统属性 → 高级 → 环境变量
```

**第三步：** 预览识别结果，确认后存入名册。不在技能库中的技能会给出校验提示，建议手动替换。

### 战斗引擎实现机制

| 机制 | 规则 |
|------|------|
| **伤害公式** | `(攻/防) × 0.9 × 技能威力 × 能力等级 × 本系加成(×1.5) × 克制倍率 × 天气加成` |
| **能力等级** | `(1 + 攻击提升 + 敌方防御降低) / (1 + 攻击降低 + 敌方防御提升)` |
| **双属性克制** | `克制倍率 = 对主属性倍率 × 对副属性倍率` |
| **性格修正** | 种族值最高项 +20%，最低项 -10%（可手动覆盖） |
| **烧伤** | 每回合 `-2% × 层数` HP，层数减半衰减，归零消失 |
| **中毒** | 每回合 `-3% × 层数` HP，层数不衰减 |
| **冻结** | 回合结束时若 HP < `maxHP × 层数/12` 则死亡 |
| **寄生** | 被寄生者每回合 -8% HP，寄生者在场时回复等量 |
| **应对** | 攻击 vs 防御/状态、防御 vs 状态 各触发对应额外效果 |
| **天气** | 雪天：所有精灵每回合 +2 冻结层；沙暴：地系技能能耗减半；雨天：水系威力 +50% |
| **换人** | 主动换人 / 精灵倒下后自动切换到首位存活精灵 |
| **能量** | 汇合聚能 +5，技能各有消耗，不足时强制聚能 |

rocom-data/
